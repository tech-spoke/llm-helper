# v1.1 アップデート

Release Date: 2025-01

## 概要

v1.1 では影響分析フェーズと、プロジェクトコンテキストの自動ロード機能を追加。

## 新機能

### Impact Analysis Phase（影響分析フェーズ）

VERIFICATIONとREADYの間に、変更の影響を実装前に分析する新しいフェーズを追加。

**機能:**
- 静的参照検出（呼び出し元、型ヒント）
- 命名規則マッチング（テスト、ファクトリ、シーダー）
- ドキュメント内キーワード検索（v1.1.1）
- 純粋HTML/CSSファイルのマークアップ緩和

**使用例:**
```
mcp__code-intel__analyze_impact
  target_files: ["app/Models/Product.php"]
  change_description: "priceフィールドの型を変更"
```

### Context Provider（コンテキストプロバイダー）

セッション開始時にプロジェクトルールと設計ドキュメントサマリーを自動ロード。

**設定 (`context.yml`):**
```yaml
project_rules:
  source: "CLAUDE.md"

document_search:
  include_patterns:
    - "**/*.md"
  exclude_patterns:
    - "node_modules/**"
```

### Markup Relaxation（マークアップ緩和）

純粋なマークアップファイル（`.html`, `.css`, `.md`）では探索要件が緩和:
- `symbols_identified`: 不要
- `find_definitions`/`find_references`: 不要
- `search_text` のみで可

**除外（ロジックを含む）:**
- `.blade.php`, `.vue`, `.jsx`, `.tsx`
